232.用栈实现队列

  1. 采用两个栈 stack1（输入栈） 和 stack2（输出栈） 进行操作。
  2. 首先声明两个全局栈，Stack<Integer> stack1; Stack<Integer> stack2; 。
  3. 在MyQueue中进行实例化。stack1 = new Stack<>(); stack2 = new Stack<>(); 。
  4. 在push数据的时候，将数据放入到stack1中即可。stack1.push(x)。
  5. 在pop数据的时候，首先判断stack2是否为空，如果不为空，则在stack2中执行弹出操作；如果为空，则将stack1中的数据全部弹出并依次压入到stack2中。
  6. 在peek（获取顶部元素）的时候，如果stack2不为空，则直接获取其顶部的元素；如果顶部元素为空，则将stack1中的元素全部弹出并依次放入stack2中，在获取stack2的顶部元素。
  7. 在empty（判断是否为空时），直接同时判断stack1和stack2是否都为空。如果都为空返回true，否则返回false。

225.用队列实现栈

  1. 采用两个队列实现一个栈的功能。
  2. 首先声明两个全局的队列，Queue<Integer> queue1;Queue<Integer> queue2; 。
  3. 在MyStack中对两个队列进行实例化。queue1 = new LinkedList<>();queue2 = new LinkedList<>(); 。
  4. 针对与queue2，它的功能就是为了实现备份。把queue1最后面的元素以外的元素都备份到queue2，然后弹出最后面的元素，再把其他元素从queue2导回queue1。
  5. 对于pop、top、empty，仅操作queue1即可。

20. 有效的括号

  • 主要是确定出现括号不匹配的情况都有哪些。

    1. 第一种情况，字符串里左方向的括号多余了 ，所以不匹配。

      • 已经遍历完了字符串，但是栈不为空，说明有相应的左括号没有右括号来匹配，所以return false。

    2. 第二种情况，括号没有多余，但是 括号的类型没有匹配上。

      • 遍历字符串匹配的过程中，发现栈里没有要匹配的字符。所以return false。

    3. 第三种情况，字符串里右方向的括号多余了，所以不匹配。

      • 遍历字符串匹配的过程中，栈已经为空了，没有匹配的字符了，说明右括号没有找到对应的左括号return false
